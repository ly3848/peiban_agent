# Supabase 连通性测试 - 需要的信息

为了完成 Supabase 连通性验证测试，我需要您提供以下信息：

## 必需信息

### 1. Supabase 项目 URL

**获取方式**:
1. 登录 [Supabase 控制台](https://app.supabase.com)
2. 选择您的项目
3. 进入 **Settings** → **API**
4. 找到 **Project URL**
5. 格式类似: `https://xxxxxxxxxxxxx.supabase.co`

**示例**: `https://abcdefghijklmnop.supabase.co`

---

### 2. Supabase API Key

**获取方式**:
1. 在同一个页面（Settings → API）
2. 找到 **Project API keys**
3. 复制 **anon public** key（用于测试）
   - 或者使用 **service_role** key（需要更高权限，但功能更完整）

**示例**: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6ImFiY2RlZmdoaWprbG1ub3AiLCJyb2xlIjoiYW5vbiIsImlhdCI6MTYxNjIzOTAyMiwiZXhwIjoxOTMxODE1MDIyfQ.xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

---

## 可选信息（用于完整测试）

### 3. Storage Bucket 名称

**获取方式**:
1. 在 Supabase 控制台，进入 **Storage**
2. 创建一个新的 Bucket（如果还没有）
3. 将 Bucket 设置为 **Public**（公开访问）
4. 记录 Bucket 名称

**示例**: `test-images`

---

### 4. Storage 测试文件路径

**准备步骤**:
1. 准备一张测试图片（建议大小 500KB 左右）
2. 上传到上面创建的 Storage Bucket
3. 记录文件的完整路径

**示例**: `test-image.jpg` 或 `images/test-image.jpg`

---

- API Key 是敏感信息，请妥善保管
- 测试完成后，可以考虑撤销测试用的 API Key
- 如果使用 service_role key，请确保不要泄露
